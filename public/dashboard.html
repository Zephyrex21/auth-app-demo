Security Info
How this app protects your data — password hashing, token verification, and the full auth flow.
Password Hashing with bcrypt

Your password is never stored as plain text. The server uses bcrypt.genSalt(10) and bcrypt.hash() to produce a one-way hash. Even if the database is compromised, passwords remain protected. A real bcrypt hash looks like this:

$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
$2a$ — bcrypt version
10$ — cost factor (2¹⁰ = 1024 rounds)
Next 22 chars — random salt (unique per hash)
Remaining — hashed password

Security Features in This App

Passwords Hashed with bcrypt

Cost factor 10 makes brute-force attacks computationally expensive. Even database leaks won't expose real passwords.

JWT Signature Verification

Every protected API call verifies the token's HMAC-SHA256 signature using the server-side secret. Tampered tokens are rejected.

Token Expiry (1 Day)

JWT tokens include an exp claim. The server rejects expired tokens automatically, limiting the window for token misuse.

Password Excluded from API Responses

The profile endpoint uses .select('-password') — your hashed password never leaves the server in any response.

401 on Invalid/Expired Tokens

The authMiddleware catches missing, malformed, or expired tokens and returns 401, preventing unauthorized access.

API Reference
All endpoints in this app — methods, auth requirements, request/response shapes.
POST /api/users/register Create a new account Public
Request Body
{
  "name":     "Saurabh Raj",
  "email":    "you@example.com",
  "password": "secret123"
}
Response 201
{
  "message": "User registered successfully",
  "user": {
    "id": "64abc123...",
    "name": "Saurabh Raj",
    "email": "you@example.com"
  }
}
POST /api/users/login Authenticate & get JWT Public
Request Body
{
  "email":    "you@example.com",
  "password": "secret123"
}
Response 200
{
  "message": "Login successful",
  "token":   "eyJhbGci..."  // JWT
}
GET /api/users/profile Fetch authenticated user Bearer Token Required
Request Headers
Authorization: Bearer eyJhbGci...
Content-Type:  application/json
Response 200
{
  "user": {
    "_id":       "64abc123...",
    "name":      "Saurabh",
    "email":     "you@example.com",
    "createdAt": "2025-06-01T..."
  }
}
The authMiddleware runs before this controller. It extracts & verifies the Bearer token via jwt.verify(token, process.env.JWT_SECRET). On success, req.user is populated with the decoded payload { id, email }.
Common Error Responses
400
Bad Request — User already exists, or invalid email/password combination
401
Unauthorized — No token provided, token is malformed, or token has expired
500
Server Error — Database connection failed, or unexpected runtime error
